# 异常消息 i18n（0.3.0 新增）

0.3.0 引入基于 `fluent-rs` 的异常消息国际化，支持中英文切换，通过 `i18n` feature 启用。

## Feature 启用

```toml
[dependencies]
bulwark = { version = "0.7", features = ["i18n"] }
```

`i18n` feature 启用 `fluent`（fluent-rs）+ `unic-langid`。

## 设计

- **`BulwarkLocale` 枚举**：`Zh`（默认）/ `En`，向后兼容 0.2.x 硬编码中文行为
- **thread_local 栈式 scope**：`set_locale()` 返回 RAII guard，drop 时自动 pop，支持嵌套
- **`OnceCell` 缓存 `FluentBundle`**：首次访问加载 `.ftl` 资源，后续零开销
- **`Display` trait 动态查询**：`BulwarkError::to_string()` 依据当前 locale 查询 fluent bundle

## 语言资源文件

位于仓库 `locales/` 目录：

`locales/zh.ftl`（默认）：

```fluent
not-login = 未登录: {$detail}
not-permission = 无权限: {$detail}
not-role = 无角色: {$detail}
invalid-token = Token 无效: {$detail}
expired-token = Token 已过期: {$detail}
dao = DAO 错误: {$detail}
config = 配置错误: {$detail}
# ... 其余错误类型
```

`locales/en.ftl`：

```fluent
not-login = Not logged in: {$detail}
not-permission = Permission denied: {$detail}
not-role = Role denied: {$detail}
invalid-token = Invalid token: {$detail}
expired-token = Token expired: {$detail}
# ... 其余错误类型
```

每条消息通过 `{$detail}` 占位符注入具体上下文。

## 切换语言

```rust
use bulwark::i18n::{set_locale, BulwarkLocale, current_locale};
use bulwark::error::BulwarkError;

// 默认中文
let err = BulwarkError::NotLogin("请先登录".to_string());
assert_eq!(err.to_string(), "未登录: 请先登录");
assert_eq!(current_locale(), BulwarkLocale::Zh);

// 切换英文（RAII guard，drop 后自动恢复）
{
    let _guard = set_locale(BulwarkLocale::En);
    assert_eq!(current_locale(), BulwarkLocale::En);
    assert_eq!(err.to_string(), "Not logged in: 请先登录");
} // guard drop，恢复中文
assert_eq!(err.to_string(), "未登录: 请先登录");
```

## 嵌套 set_locale

栈式 scope 支持嵌套调用：

```rust
let _g1 = set_locale(BulwarkLocale::En);
{
    let _g2 = set_locale(BulwarkLocale::Zh);  // 临时切回中文
    assert_eq!(current_locale(), BulwarkLocale::Zh);
} // g2 drop，恢复英文
assert_eq!(current_locale(), BulwarkLocale::En);
```

## 公开 API

i18n 模块对外暴露以下 API：

| API | 说明 |
|:---|:---|
| `BulwarkLocale` 枚举 | `Zh`（默认）/ `En` |
| `current_locale()` | 返回当前线程的 locale |
| `set_locale(locale)` | 设置当前线程的 locale，返回 RAII guard |
| `translate_error(&BulwarkError)` | 将 `BulwarkError` 翻译为当前 locale 字符串 |
| `translate_detail(key, &[(&str, &str)])` | 按 FTL key + args 翻译（不依赖 `BulwarkError`），供 `loc!` 宏使用 |
| `loc!` 宏 | `loc!("key", "fallback", ("arg_k", "arg_v"))` → 委托 `translate_detail` |

`BulwarkLocale::as_lang_id()` 为内部私有函数（构造 `FluentBundle` 时匹配资源用），不对外暴露。

## loc! 宏

`loc!` 宏用于在不依赖 `BulwarkError` 的场景按 key 翻译（如社交登录 / Keycloak 模块的 detail 字段）：

```rust
use bulwark::loc;

// 无参数
let msg = loc!("wechat-error-response", "WeChat error".to_string());

// 带参数
let msg = loc!(
    "wechat-error-response",
    "WeChat error".to_string(),
    ("code", "40029"),
    ("message", "invalid code")
);
```

宏签名：`loc!($key:expr, $fallback:expr $(, ($arg_k:expr, $arg_v:expr))*)`。
`$fallback` 参数兼容旧式 detail 串，使调用方无需即时改造即可使用。

## ICU4X 增强层（`i18n-icu` feature）

启用 `i18n-icu` feature 后，模块额外提供 `icu_enhanced` 子模块（基于 ICU4X）：

```toml
[dependencies]
bulwark = { version = "0.7", features = ["i18n", "i18n-icu"] }
```

| API | 说明 |
|:---|:---|
| `plural_category(count)` | 返回当前 locale 下的复数类别（Cardinal，如 One/Other） |
| `format_datetime_locale(&DateTime)` | 格式化 chrono `DateTime` 为本地化字符串（替代 `{:?}` Debug 格式） |
| `format_number_locale(n)` | 格式化整数为本地化字符串（千分位 / 数字系统） |

仅在 `i18n-icu` feature 启用时编译，不影响现有翻译逻辑。

## Display trait 动态查询

`BulwarkError` 的 `Display` 实现调用 `translate_error(&self)`，依据 `current_locale()` 选择 bundle：

- `Zh` → 加载 `zh.ftl`，输出中文
- `En` → 加载 `en.ftl`，输出英文
- bundle 经 `OnceCell` 缓存，首次加载后零开销查询

## 注意事项

- 默认 `Zh`，未调用 `set_locale` 时行为与 0.2.x 完全一致（向后兼容）
- locale 是 **线程本地** 的，需在每个工作线程显式设置（web 中间件可在请求入口统一设置）
- `{$detail}` 内容本身不翻译（业务方提供的上下文），仅翻译框架错误前缀
- `translate_error` 内部对结构化 detail（`key::arg0::arg1`）会优先按 key 翻译，找不到则回退到 variant 默认 key
